## 1. 产品定位与愿景

### 1.1 一句话定位

**给孩子一个 24 小时在线的 AI 家庭教师，给家长一份随时可查的学习报告。**

### 1.2 产品愿景

在中国家庭中，辅导孩子作业是家长最焦虑的场景之一：
- 工作繁忙，下班后没有精力辅导
- 知识遗忘，高年级题目无力讲解
- 方法不当，辅导变成争吵
- 沟通缺失，不了解孩子真实学习状况

我们的产品是一个部署在家中的 AI 教学设备（RK3576 边缘计算盒子），通过微信和 Web 浏览器两个入口，让孩子随时获得苏格拉底式引导教学，让家长远程了解学习进展。

### 1.3 核心价值主张

| 角色 | 痛点 | 解决方案 | 价值 |
|------|------|----------|------|
| **孩子** | 作业不会做没人教 / 刷题枯燥 / 错题反复错 | AI 教师苏格拉底式引导 / 自动出同类题巩固 / 知识点关联讲解 | 学习有陪伴，进步看得见 |
| **家长** | 没时间辅导 / 不知道孩子学得怎样 / 想帮帮不上 | 微信拍题即可辅导 / 日报周报月报 / 薄弱点可视化 | 工作育儿两不误 |
| **家庭** | 辅导班贵 / 效果难评估 / 孩子抵触 | 硬件一次购买，全家免费用 / AI 耐心教学不吼叫 | 低成本高质量教育 |

### 1.4 角色定位模型

```
                      ┌─────────────────────────┐
                      │      家长 (管理者)        │
                      │  设定目标 · 查看报告 · 管控   │
                      │  入口：微信 (首要) + Web   │
                      └───────────┬─────────────┘
                                  │ 授权 & 监督
                                  ▼
┌────────────────────────────────────────────────────────┐
│                      AI 教师 (执行者)                    │
│  苏格拉底引导 · 自动出题 · 掌握度追踪 · 错题归因 · 间隔复习 │
└────────┬────────────────────────────────────┬──────────┘
         │ 教学 & 反馈                        │ 互动 & 提问
         ▼                                    ▼
┌─────────────────────┐            ┌─────────────────────┐
│   孩子 A (学习者)    │            │   孩子 B (学习者)    │
│  独立画像 · 独立进度   │            │  独立画像 · 独立进度   │
│  入口：微信 + Web     │            │  入口：微信 + Web     │
└─────────────────────┘            └─────────────────────┘
```

**核心原则：家长管、AI 教、孩子学** — 三方各司其职。

## 2. 目标用户画像

### 2.1 孩子 — 学习者

#### 小学段 (6-12 岁)

| 维度 | 描述 |
|------|------|
| **年龄** | 6-12 岁，小学 1-6 年级 |
| **能力** | 识字量有限，抽象思维正在发展 |
| **痛点** | 作业不会做不敢问 / 注意力易分散 / 需要即时反馈 |
| **使用方式** | 微信发送作业照片 → 语音/文字回复 |
| **教学偏好** | 短互动 (单次 < 5 分钟)、图片优先、鼓励式反馈 |
| **技术门槛** | 几乎为零 — 微信拍照聊天 |

**典型用户：**
- **小明，8 岁，三年级** — 每天放学后通过微信发送数学作业照片，AI 教师逐题引导。不会的音视频通话问奶奶，自己用微信语音输入答案。

#### 初中段 (12-15 岁)

| 维度 | 描述 |
|------|------|
| **年龄** | 12-15 岁，初中 1-3 年级 |
| **能力** | 逻辑思维发展期，自学能力初步形成 |
| **痛点** | 科目增多、难度陡增、错题整理耗时 |
| **使用方式** | 微信发题 + Web 深度学 |
| **教学偏好** | 多轮互动、错题自动归类、薄弱点专项训练 |
| **技术门槛** | 低 — 可独立操作 Web 界面 |

**典型用户：**
- **小丽，14 岁，初二** — 通过微信发作业照片，AI 引导解题。周末在 Web 端做 AI 自动生成的薄弱点强化试卷。主动查看自己的错题本和知识点掌握度雷达图。

### 2.2 家长 — 管理者

| 维度 | 描述 |
|------|------|
| **年龄** | 30-45 岁，在职工作 |
| **身份** | 80/90 后，大学学历为主 |
| **痛点** | 工作时间与孩子学习时间错位 / 无力辅导高年级 / 不知道孩子真正的薄弱点 |
| **使用方式** | 微信接收报告 + 偶尔远程发题 |
| **核心诉求** | 知道孩子今天学了吗、学得怎么样、哪里需要加强 |
| **技术门槛** | 低 — 微信聊天即可 |

**典型用户：**
- **王妈妈，35 岁，上班族** — 下班路上收到 AI 推送的今日学习报告。看到孩子分数应用题正确率只有 40%，在微信里说"出几道分数应用题的练习"。浏览器的绑定页面帮孩子开通了专属账号。
- **李先生，38 岁，经常出差** — 人在外地，通过微信了解孩子每天的作业情况。收到周报发现孩子几何部分连续薄弱，远程让 AI 加强几何专题训练。

### 2.3 多子女家庭场景

| 场景 | 产品应对 |
|------|----------|
| 两个孩子不同年级 | 独立 learner_id，独立学习进度 |
| 怕孩子用 AI 抄答案 | 苏格拉底式引导 — AI 不直接给答案，用提问引导 |
| 不同孩子不同学科 | 独立学科掌握度追踪 |
| 家长微信群互通 | 一次设备购买，全家孩子共享 (最多 5 个独立学习者) |

### 3.2 双入口策略

| 维度 | 微信 (手机) | Web 浏览器 (电脑/平板) |
|------|------------|----------------------|
| **定位** | 日常轻量辅导 | 深度学习+数据洞察 |
| **设备** | 家长/孩子手机 | 家中电脑 / 学习平板 (如 iPad) |
| **孩子年龄** | 全年龄段 (6-15岁) | 偏大龄 (10-15岁，能独立操作电脑) |
| **典型使用时段** | 每日放学后，碎片时间 | 周末/假期，30分钟以上专注时段 |
| **交互方式** | 拍照 + 文字/语音回复 | 键盘输入 + 鼠标点击 + 可视化界面 |
| **核心功能** | OCR 发题 → 引导教学 → 自动下一题 | 学习仪表盘 → 错题本 → 专项练习 → 模拟考 |
| **数据呈现** | 纯文本，微信消息格式 | 图表、雷达图、柱状图、统计表格 |
| **操作深度** | 浅 (每次 5-15 分钟) | 深 (每次 30-60 分钟) |
| **干扰因素** | 微信消息通知可能打断 | 专注学习无干扰 |
| **家长偏好** | ❌ 不愿孩子长时间用手机 | ✅ 电脑屏幕大、护眼、可监督 |

**双入口关系图：**

```
微信端发题 → 数据记录到掌握度
                    │
                    ▼
           Web 端展示学习数据
                    │
                    ├─ 查看统计 → 发现薄弱点
                    ├─ 查看错题 → 针对性复习
                    ├─ 生成练习 → 深度巩固
                    └─ 生成试卷 → 模拟考试
                    │
                    ▼
           Web 端完成的数据同步回掌握度
                    │
                    ▼
           微信端后续教学覆盖新知识点
```

### 3.2 完整场景分解

#### 场景 A：日常作业辅导 (高频、核心)

```
触发条件: 孩子在微信发送图片 (作业/试卷)
用户角色: 孩子
频率: 每天 1-3 次
期望时长: 每次 5-15 分钟

流程:
1. 孩子打开微信，拍照发送作业页面
2. AI 接收图片，后台 OCR 识别题目内容
3. 【教育内容检测】判断是否为学习材料
   - 是: 自动进入教学流程
   - 否: 告知无法识别，请重拍
4. AI 识别题目类型：
   - 整份试卷 → 逐题引导模式
   - 单道题目 → 单题引导模式
   - 知识点提问 → 直接讲解模式
5. AI 展示第 1 题 + 引导问题 (不直接给答案)
6. 孩子回复答案 (文字/语音)
7. AI 判断对错 + 知识点讲解 + 自动下一题
8. 答题结果记录到掌握度数据
9. 如连续答错 → 自动出同类巩固题
10. 如积累 ≥3 个薄弱点 → 后台生成强化试卷 (24h 冷却)

成功标准:
✓ 孩子独立完成作业辅导，不需要家长介入
✓ 每道题孩子有思考过程 (不直接给答案)
✓ 错题自动记录并安排复习
✓ 单次交互不超过 15 分钟 (符合儿童注意力)
```

#### 场景 B：家长远程了解 (高频、核心)

```
触发条件: 家长主动询问或定时推送
用户角色: 家长
频率: 每天 1 次看日报，每周 1 次看周报
期望时长: 30 秒浏览

流程:
1. 晚上 20:30，家长微信收到日报推送
2. 摘要包含：答题数、正确率、薄弱知识点
3. 家长快速浏览，了解今日学习情况
4. 如发现薄弱点 → 在微信中回复"出几道 XX 的题"
5. AI 收到家长指令 → 生成针对性练习
6. 练习推送到孩子微信

成功标准:
✓ 家长 30 秒内了解孩子今日学习状况
✓ 家长不需要打开 App，微信内完成所有操作
✓ 家长可随时发起"加练"指令
```

#### 场景 C：薄弱点强化训练 (中频)

```
触发条件: 系统检测到薄弱知识点累计 ≥3 个
用户角色: 孩子 + AI (自动触发)
频率: 每 24h 最多 1 次
期望时长: 20-30 分钟

流程:
1. 系统检测到 3+ 薄弱点 (掌握度 < 0.6)
2. 后台调用 LLM 生成强化试卷 (选择题+填空题+解答题)
3. 在下次教学交互时注入试卷到 SOUL.md
4. 引导孩子完成试卷
5. 答题结果更新掌握度
6. 如掌握度提升 → 移出薄弱点列表

成功标准:
✓ 针对性覆盖 ALL 薄弱知识点
✓ 难度分层 (基础 50% + 中等 35% + 拔高 15%)
✓ 孩子完成后薄弱点掌握度明显提升
```

#### 场景 D：考前系统复习 (低频)

```
触发条件: 家长主动要求/孩子主动要求
用户角色: 家长 + 孩子
频率: 考试前
期望时长: 30-60 分钟

流程:
1. 家长在微信中发送"出期末考试卷"
2. AI 读取所有已学知识点 + 错题记录
3. 生成完整模拟试卷 (含答案)
4. 孩子逐题作答
5. AI 批改 + 知识点归因
6. 生成考前复习报告

成功标准:
✓ 覆盖本学期主要知识点
✓ 包含错题同类变式题
✓ 输出复习建议
```

#### 场景 E：Web 端深度学习 (高频、核心)

```
触发条件: 孩子在电脑/平板上打开浏览器访问设备 Web UI
用户角色: 孩子 (独立操作)
使用设备: 电脑 / 学习平板 (非手机)
频率: 每周 2-3 次 (周末为主)
期望时长: 每次 30-60 分钟
核心价值: 这是孩子深度学习的核心场景，弥补微信轻交互不可替代的体验

场景动机:
  - 家长不希望孩子长时间看手机 (屏幕小、伤眼、易分心)
  - 微信不适合展示复杂的学习数据 (图表、知识点关系、趋势)
  - 系统性的复习需要专注的时段，碎片化的微信交互不够
  - 孩子需要独立的学习空间，不被微信聊天消息打断

流程:
1. 孩子在电脑浏览器输入设备地址，进入学习仪表盘
2. 仪表盘展示：
   - 今日概览 (今日答题数、正确率、学习时长)
   - 知识点掌握度雷达图 (全部已学知识点的水平一目了然)
   - 周学习趋势折线图 (7 天正确率变化)
   - 薄弱知识点列表 (掌握度 < 0.6，按从低到高排序)
   - 近期错题 (最近 10 道错题，按时间倒序)
3. 孩子可执行以下操作：
   a. 点击薄弱知识点 → 查看详情 → 生成专项练习 (3 道)
   b. 点击错题 → 查看题目详情 + 正确答案 + 讲解
   c. 按知识点筛选错题本 → 针对性复习
   d. 生成强化试卷 → 完整作答 (选择题/填空题/解答题三段式)
   e. 生成学习报告 (日报/周报/月报) → 查看/打印
   f. 查看答题历史 → 按时间/知识点/对错筛选
4. Web 上完成的所有练习/试卷结果同步回掌握度数据库
5. 下次微信教学时，AI 自动识别已掌握的知识点，调整教学重点

成功标准:
  ✓ 学习仪表盘打开 < 2s
  ✓ 所有数据展示为可视化图表 (不依赖第三方服务)
  ✓ 生成练习/试卷 < 15s
  ✓ 所有操作在浏览器内完成，不需要扫码或跳转
  ✓ 全部数据本地存储，不依赖外部服务器
```

#### 场景 F：考前系统复习 (低频)

```
触发条件: 期中/期末考试前，家长或孩子主动发起
用户角色: 家长 + 孩子
使用设备: Web 浏览器 (电脑/平板)
频率: 每学期 2-3 次 (期中、期末)
期望时长: 60-90 分钟

流程:
1. 家长在微信中说"出期末考试卷" 或 孩子在 Web 端点击"模拟考试"
2. AI 读取所有已学知识点 + 全部错题记录
3. 生成完整模拟试卷 (选择题 + 填空题 + 解答题，附答案)
4. 孩子登录 Web 端逐题作答
5. AI 批改 + 每题知识点归因
6. 生成考前复习报告：
   - 已掌握知识点 (绿色)
   - 需要复习的知识点 (黄色)
   - 薄弱知识点 (红色，需重点复习)
7. 建议复习计划 (按 Ebbinghaus 间隔到考试前的复习安排)

成功标准:
  ✓ 覆盖本学期主要知识点
  ✓ 包含错题同类变式题 (不低于 30%)
  ✓ 输出可打印的复习报告
```

## 5. 教学科学基础

### 5.1 教学法模型

系统融合三种经典教学理论：

```
┌──────────────────────────────────────────────────────────┐
│                   三层教学模型                             │
│                                                          │
│  第一层：苏格拉底式引导 (每次交互)                         │
│  ┌────────────────────────────────────────────────────┐  │
│  │  展示题目 → 引导提问 → 孩子回答 → 判断对错 → 讲解  │  │
│  │  ↕                                                 │  │
│  │  核心原则：不直接给答案，用问题激发思考               │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  第二层：分层引导策略 (同题多轮)                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │  第一层：方向引导（开放性问题，指向知识点方向）       │  │
│  │  第二层：方法提示（关键步骤提示，缩小思考范围）       │  │
│  │  第三层：完整解析（展示解题过程，出巩固题）           │  │
│  │  ↕ 每层之间等待学生回答，不跳跃                      │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  第三层：艾宾浩斯间隔复习 (跨时间维度)                    │
│  ┌────────────────────────────────────────────────────┐  │
│  │  掌握度 < 0.4 → 次日复习                            │  │
│  │  掌握度 0.4-0.6 → 3 天复习                          │  │
│  │  掌握度 0.6-0.8 → 7 天复习                          │  │
│  │  掌握度 0.8-0.9 → 14 天复习                         │  │
│  │  掌握度 ≥ 0.9 → 30 天复习                           │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

### 5.2 掌握度模型

```
┌─────────────────────────────────────────────────┐
│                 掌握度演算规则                     │
│                                                   │
│  每个知识点 (kp_id) 独立追踪:                     │
│    level = correct_count / total_count            │
│    level ∈ [0.0, 1.0]                             │
│                                                   │
│  分类:                                            │
│    level ≥ 0.9  → 已巩固 (绿色)                   │
│    level ≥ 0.8  → 已掌握 (蓝色)                   │
│    level ≥ 0.6  → 进步中 (黄色)                   │
│    level ≥ 0.4  → 学习中 (橙色)                   │
│    level < 0.4  → 薄弱 (红色)                     │
│                                                   │
│  薄弱点定义: level < 0.6                         │
│  复习触发: schedule_review() 按 Ebbinghaus 间隔    │
└─────────────────────────────────────────────────┘
```

### 5.3 答案评估格式

DT 教学引擎在回复末尾附加结构化标记：

```
[ANSWER:correct:学科/章节/知识点ID]
[ANSWER:wrong:学科/章节/知识点ID]
```

示例：
```
第3题 答对了！
【知识点：幂的运算】
同底数幂相乘，底数不变指数相加。a²·a³ = a^(2+3) = a⁵。

[ANSWER:correct:数学/代数/幂的运算]
```

平台处理：
1. 剥离标记 (用户不可见)
2. 更新掌握度
3. 记录答题历史
4. 安排间隔复习

### 5.4 错题归因模型

```
学生答错后，AI 判断错因类型 → 针对性讲解：

┌─────────────┐    学生答错
│  错因分类    │
└──────┬──────┘
       │
       ├─ 概念不清 → 回归基本概念 + 生活类比
       │
       ├─ 计算失误 → 指出具体步骤错误，不重讲概念
       │
       ├─ 审题问题 → 引导重新读题，标注关键条件
       │
       └─ 知识混淆 → 对比区分两个相似概念的异同
```
